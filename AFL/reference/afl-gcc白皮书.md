# 设计声明

AFL 尽可能不专注在单个原理操作和特定某个理论证明，这个工具可以认为是hacks工具而且已经在实践中测试，高效，且以最简单的方式实现，当时我能想到最健壮的方式。



得益于为该工具服务的轻量级指令使得很多最终功能变得可能，但是这个机制应该仅仅被认为是达到目的的一种手段。唯一的主要原则，速度，可靠性和易用性。

# 覆盖测量

指令注入到编译程序来捕捉覆盖分支 ，以及粗略分支命中技术。在分支点注入的代码在本质等同与

```c
  cur_location = <COMPILE_TIME_RANDOM>;
  shared_mem[cur_location ^ prev_location]++; 
  prev_location = cur_location >> 1;
```



cur_location 是随机生成的来简化链接复杂项目的过程和保持异或输出均匀分布。

shared_mem[] 是一块64kb大小的共享内存区域用来被调用者传递给已装备（应该是指使用afl配套的编译工具编译过）的二进制文件。  

映射的大小是被指定的所以碰撞在几乎所有目标都是零星的，通常为2k到10k的可发现的分支点

| Branch cnt  |  Colliding tuples | Example targets|
| ---- | ---- | ---- |
|        1,000 | 0.75%            | giflib, lzo|||||
|        2,000 | 1.5%             | zlib, tar, xz||||
|        5,000 | 3.5%             | libpng, libwebp|||
|       10,000 | 7%               | libxml||
|      20,000 | 14%              | sqlite|
|     50,000 | 30%              | -|

同时映射的尺寸足够小到允许以微秒级来分析在接受端并且可以轻松的放入L2 缓存。

这种形式的覆盖范围在程序执行路径中提供了相当多的了解相比与简单的块覆盖，尤其是烦琐的区分以下执行追踪

```
  A -> B -> C -> D -> E (tuples: AB, BC, CD, DE)
  A -> B -> D -> C -> E (tuples: AB, BD, DC, CE)
```

这有助于发现基础代码中的细微故障条件，因为安全漏洞通常与意外事件相关联或错误的状态转换，而不仅仅是到达一个新的基本块。
